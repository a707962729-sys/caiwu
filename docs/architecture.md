# 财务管家系统架构设计

## 一、系统概述

财务管家是一款面向中小企业的智能财务管理系统，支持本地或云端部署，提供移动端访问。

## 三、用户角色与权限

### 3.1 角色定义

| 角色 | 权限级别 | 功能范围 |
|------|---------|---------|
| **老板 (boss)** | 最高级 | 全部数据查看、审批、报表、员工管理 |
| **会计 (accountant)** | 中级 | 记账、票据、报表、账目核对 |
| **员工 (employee)** | 基础 | 报销申请、票据上传、查询个人数据 |

### 3.2 权限矩阵

| 功能模块 | 老板 | 会计 | 员工 |
|---------|------|------|------|
| 记账管理 | ✓✓ | ✓✓ | ✗ |
| 报销审批 | ✓✓ | ✓ | ✗ |
| 合同管理 | ✓✓ | ✓ | ✓(查看) |
| 财务报表 | ✓✓ | ✓✓ | ✗ |
| 员工管理 | ✓✓ | ✗ | ✗ |
| 个人中心 | ✓ | ✓ | ✓✓ |
| AI助手 | ✓✓ | ✓✓ | ✓ |

### 4.2 存储空间设计

| 存储类型 | 说明 | 访问权限 |
|---------|------|---------|
| **共享存储** | 项目代码、数据库、上传文件 | 所有 Agent |
| **独立记忆** | 各 Agent 的 MEMORY.md | 仅对应 Agent |
| **私有文件** | 敏感数据（工资、合同） | 按权限控制 |

## 七、老板端功能

### 7.1 数据看板

| 模块 | 内容 |
|------|------|
| **财务概览** | 收入/支出/利润/现金流 |

### 7.2 员工数据展示

```typescript
interface EmployeeDashboard {
  // 基本信息
  employee: {
    id: number
    name: string
    department: string
    position: string
  }
  
  // 考勤数据
  attendance: {
    workDays: number      // 出勤天数
    leaveDays: number     // 请假天数
    lateTimes: number     // 迟到次数
    overtimeHours: number // 加班时长

  // 薪资数据

    base: number          // 基本工资
    bonus: number         // 奖金

  // 报销统计
  reimbursements: {
    total: number         // 报销总额
    pending: number       // 待审批
    approved: number      // 已批准
  }
}
```

## 八、安全设计

### 8.1 数据安全

- 所有通信使用 HTTPS
- 敏感数据加密存储
- 审计日志记录所有操作

### 8.2 访问控制

- 基于角色的权限控制 (RBAC)
- API 请求频率限制
- 敏感操作二次验证

## 九、扩展性

### 9.1 模块化设计

- 认证模块可独立部署
- Agent 可动态扩展
- 支持多租户架构

### 9.2 集成能力

- 支持对接企业微信/钉钉
- 支持对接财务软件
- 支持对接银行系统
